# Exam Management System

This is a comprehensive web application for creating, managing, and taking multiple-choice exams. It's built with React, TypeScript, and Tailwind CSS, featuring distinct roles for administrators and students.

## Features

### Administrator Role
- **Exam Management:**
  - Create new exams with a title, date, time, and optional class/description.
  - Add, edit, and delete multiple-choice questions for each exam.
  - View all created exams in a card-based layout.
  - Enable or disable exams to control student access.
  - Assign specific students to an exam.
  - Delete exams.
- **Student Management:**
  - Create new student accounts with required (name, password) and optional details.
  - View all registered students.
  - Edit student information.
  - View exams assigned to a specific student.

### Student Role
- **Dashboard:**
  - View a list of all assigned exams.
  - See the status of each exam (Available, Taken, or Scheduled).
- **Taking Exams:**
  - Start available exams.
  - Answer multiple-choice questions (only one selection per question).
  - Submit the exam once all questions are answered.
- **Results:**
  - If an exam has already been taken, the student can see their final score.
  - If an exam is not yet available, a message shows its scheduled date and time.

## Tech Stack
- **Frontend:** React 18+, TypeScript
- **Styling:** Tailwind CSS
- **Data:** Pluggable data layer.
  - `data.ts`: Mock data layer simulating a backend (default).
  - `data_supabase.ts`: Implementation using Supabase for a real backend.

## Backend Setup (Supabase)

This project includes a Supabase implementation to serve as a live backend. To use it, you'll need to create a Supabase project and set up the database tables.

1.  **Follow `firebase-setup.txt`:** The steps for creating a project and getting API keys are similar for Supabase. Create a new project at [supabase.com](https://supabase.com).
2.  **Get API Keys:** In your Supabase project, go to **Project Settings > API** and find your **Project URL** and `anon` `public` key.
3.  **Configure `data_supabase.ts`:** Open the `data_supabase.ts` file and replace the placeholder credentials with your URL and key.
4.  **Create Tables:** Go to the **Table Editor** in your Supabase dashboard and run the following SQL queries to create the necessary tables.

    ```sql
    -- Create the users table to store public profile information
    -- This links to the private auth.users table
    CREATE TABLE public.users (
      id uuid NOT NULL,
      password text,
      username text NOT NULL,
      role text NOT NULL,
      "firstName" text NOT NULL,
      "lastName" text NOT NULL,
      age integer,
      "documentNumber" text,
      phone text,
      address text,
      CONSTRAINT users_pkey PRIMARY KEY (id),
      CONSTRAINT users_id_fkey FOREIGN KEY (id) REFERENCES auth.users (id) ON DELETE CASCADE
    );
    ALTER TABLE public.users ENABLE ROW LEVEL SECURITY;

    -- Create the exams table
    CREATE TABLE public.exams (
      id text NOT NULL DEFAULT ('exam_'::text || (random()::text)),
      title text NOT NULL,
      class text,
      isActivo boolean,
      description text,
      "dateTime" timestamptz NOT NULL,
      questions jsonb NOT NULL,
      "isEnabled" boolean NOT NULL,
      "allowedStudentIds" uuid[] NOT NULL,
      CONSTRAINT exams_pkey PRIMARY KEY (id)
    );
    ALTER TABLE public.exams ENABLE ROW LEVEL SECURITY;

    -- Create the submissions table
    CREATE TABLE public.submissions (
      id bigint generated by default as identity,
      "examId" text NOT NULL,
      "studentId" uuid NOT NULL,
      answers jsonb NOT NULL,
      score real NOT NULL,
      "submittedAt" timestamptz NOT NULL,
      CONSTRAINT submissions_pkey PRIMARY KEY (id),
      CONSTRAINT submissions_examId_fkey FOREIGN KEY ("examId") REFERENCES exams(id) ON DELETE CASCADE,
      CONSTRAINT submissions_studentId_fkey FOREIGN KEY ("studentId") REFERENCES users(id) ON DELETE CASCADE
    );
    ALTER TABLE public.submissions ENABLE ROW LEVEL SECURITY;
    ```
5. **Update `App.tsx`:** To switch from the mock data to Supabase, open `App.tsx` and change the import from `import { api } from './data';` to `import { api } from './data_supabase';`.

## Project Structure
- `index.html`: Main HTML entry point with Tailwind CSS.
- `index.tsx`: React application root.
- `App.tsx`: Core component managing authentication and routing between roles.
- `types.ts`: All TypeScript type definitions.
- `data.ts`: Mock database and API functions.
- `data_supabase.ts`: API functions for a Supabase backend.
- `components/`: Contains all UI components and views, separated logically.
- `pages/`: Contains the main page components for different user roles.
- `firebase-setup.txt`: Instructions for setting up a real Firebase backend.
- `firebase-rules.txt`: Recommended Firestore security rules for this application.
